Develop a complete web-based application called “Hacker vs System”, designed as an educational and visually immersive information security project. The application must simulate real-world cybersecurity scenarios, demonstrate vulnerabilities, allow exploitation using security tools, and implement secure fixes.

The website must not look like a basic academic form-based system. Instead, it should have a modern, hacker-style, cybersecurity-themed interface with a professional and engaging user experience.

🔹 Technology Requirements
Backend: Python using Flask
Frontend: HTML, CSS, Bootstrap + custom CSS
Database: SQLite
Security Testing Tool Integration: Burp Suite
Cryptography Demonstration Tool: CrypTool
🔹 UI / UX DESIGN REQUIREMENTS (VERY IMPORTANT)

The interface must follow a cybersecurity / hacker aesthetic:

Dark theme (black / deep gray background)
Neon accents (green, blue, red)
Monospace or terminal-style fonts
Dashboard styled like a security control panel
Animated elements (glow effects, hover highlights, transitions)
Visual Concepts:
Terminal-like panels
Glowing buttons (e.g., “ACCESS GRANTED”, “BREACH SYSTEM”)
Status indicators:
🔴 Vulnerable
🟢 Secure
Sections styled like:
“System Logs”
“Security Layers”
“Access Control Matrix”

The UI should feel like:

a hacking simulation dashboard or cybersecurity lab environment

🔹 Core Functional Requirements
1. Authentication System
User registration and login
Session management
Logout functionality
2. Role-Based Access Control (RBAC)
Roles:
Admin
User
Restrictions:
/dashboard → all users
/admin → admin only
Role validation must be enforced
3. Mode System (Core Feature)

Implement a toggle:

🔴 Insecure Mode (Attack Mode)
🟢 Secure Mode (Defense Mode)

UI should visually reflect the mode:

Color changes
Labels like:
“System Compromised”
“System Hardened”
🔹 Security Levels (Gamified System)

Design the application as levels or missions, each representing a vulnerability:

🔓 Level 1: Weak Authentication

Insecure Mode:

Plain text password storage

Secure Mode:

Password hashing (SHA-256 or bcrypt)
👑 Level 2: Broken Access Control

Insecure Mode:

Role stored in client-side cookies

Secure Mode:

Role stored and verified on server
🧨 Level 3: Parameter Tampering

Insecure Mode:

No input validation

Secure Mode:

Input validation and restrictions
🍪 Level 4: Session Hijacking

Insecure Mode:

Weak session handling

Secure Mode:

Secure cookies + expiration
