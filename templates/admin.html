ATTACK MODE — Client-Side Role

Role is read from the role cookie. An attacker can use browser DevTools or Burp Suite to change the cookie value to admin and bypass access controls entirely.

document.cookie = "role=admin"
SECURE MODE — Server-Side Permissions

Role is stored in the server-side session only. Each route checks permissions from ROLE_PERMISSIONS map. Cookie tampering has zero effect on authorization.

ROLE_PERMISSIONS['admin'] = ['access_admin', ...]
