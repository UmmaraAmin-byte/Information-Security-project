ENCRYPTION VS HASHING
HASHING
  • One-way — irreversible
  • Fixed output length
  • Same input → same hash
  • Used for: passwords, checksums
  • bcrypt, SHA-256, Argon2
password → $2b$10$...
ENCRYPTION
  • Two-way — reversible
  • Variable output length
  • Requires a key
  • Used for: data at rest/transit
  • AES-256, RSA, TLS
plaintext + key → ciphertext
DIGITAL SIGNATURE CONCEPT
1
HASH THE MESSAGE
SHA-256("{{ sample }}")
{{ sha256[:32] }}...
2
SIGN WITH PRIVATE KEY
RSA/ECDSA(hash, privateKey)
→ digital signature bytes
3
VERIFY WITH PUBLIC KEY
verify(sig, hash, publicKey)
→ TRUE — message authentic
Digital signatures prove authenticity (who signed it) and integrity (not tampered). JWT uses HMAC-SHA256 as a symmetric signature.
