VULNERABILITY DETAILS
INSECURE (Attack Mode)
  • No HttpOnly — JavaScript can read session cookie
  • No SameSite — CSRF attacks work
  • No expiry — permanent hijack window
  • No IP binding — token reuse from other IPs
  • Session not cleared on login — fixation risk
SECURE (Defense Mode)
  • HttpOnly flag — JS cannot access cookie
  • SameSite=Strict — blocks cross-site sends
  • Expires after 30 minutes of inactivity
  • Session bound to login IP address
  • Session.clear() on login (prevents fixation)
