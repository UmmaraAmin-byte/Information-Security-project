XSS ATTACK TYPES

Reflected XSS: Payload in URL/form field, reflected back in same response. Server returns input unescaped. Used for phishing — attacker sends crafted URL.

Stored XSS: Payload stored in database (e.g. comment field), executes for every user who views it. Most dangerous type — affects all visitors.

DOM-based XSS: Payload manipulates the DOM via JavaScript without server involvement. document.write(location.hash) is a classic example.

Real impact: British Airways (2018) — 22 lines of XSS stole 380,000 payment cards. Samy worm (2005) — spread across MySpace infecting 1M accounts in 20 hours.

XSS DEFENSES
HTML ESCAPING

Convert special chars: <→&lt;, >→&gt;, etc. Jinja2 does this automatically with {{ '{{ var }}' }} — use | safe only for trusted HTML.

CONTENT SECURITY POLICY

HTTP header that tells browser which scripts are allowed: Content-Security-Policy: script-src 'self' — blocks inline scripts and external sources entirely.

HTTPONLY COOKIES

Even if XSS executes, HttpOnly prevents document.cookie from reading session tokens. This cuts off the most common XSS goal — session theft.
