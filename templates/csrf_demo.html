HOW CSRF WORKS
ATTACK SCENARIO
  1. Victim is logged into banking site
  2. Attacker hosts malicious page with hidden form
  3. Victim visits attacker's page
  4. Hidden form auto-submits to banking site
  5. Browser sends victim's session cookie automatically
  6. Bank executes the forged transaction!
DEFENSE MECHANISM
  1. Server generates unique random token per session
  2. Token embedded in every form as hidden field
  3. On POST, server validates token matches session
  4. Forged requests from other sites lack valid token
  5. Request rejected — no token = no action
  6. Also: SameSite=Strict cookie flag prevents cross-origin sends
